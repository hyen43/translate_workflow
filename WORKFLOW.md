# 워크플로우 상세 레퍼런스

> 사용 개요와 배포 방법은 [README.md](./README.md) 참고. 이 문서는 규칙·정책·트러블슈팅 레퍼런스입니다.

---

## 동작 흐름

```
[입력] Figma URL + Confluence URL (매번 입력 — 저장 없음)
   ↓
[표 확인] Confluence 페이지에서 "한국어" 컬럼이 있는 첫 번째 표 탐지
   ↓ 헤더를 읽어 언어 컬럼 목록 표시 → 번역할 언어 체크
[실행]
   ↓ Figma 파일 전체 페이지에서 Trans* 프레임 텍스트 수집
   ↓ 중복 제거 후 기존 표(전체 언어 컬럼)와 비교 → 신규만 표 뒤에 추가
   ↓ 체크된 언어의 빈 셀(신규 + 기존)을 Claude(Haiku)가 번역
   ↓ 원문이 영어면 → 영어 셀에 원문 보존 + 한국어 셀에 한국어 번역
   ↓ 컬럼 추가 + 행 추가 + 셀 채움을 한 번의 저장으로 반영
[검수] 사람이 Confluence에서 직접 수정
```

---

## 토큰 발급

`.env`(로컬) 또는 Streamlit Cloud Secrets(배포)에 아래 값이 필요합니다.

| 키 | 발급 위치 |
|---|---|
| `FIGMA_TOKEN` | Figma → Settings → Security → Personal access tokens |
| `CONFLUENCE_EMAIL` | 대상 Confluence 페이지에 **편집 권한**이 있는 Atlassian 계정 이메일 |
| `CONFLUENCE_TOKEN` | id.atlassian.com → Security → API tokens → **Create API token** |
| `ANTHROPIC_API_KEY` | console.anthropic.com → API Keys |
| `APP_PASSWORD` | 팀에서 정한 접속 비밀번호 (로컬 개발 시 생략 가능) |

> ⚠️ Confluence 토큰은 반드시 일반 **Create API token** (Classic).
> `Create API token with scopes` (Scoped) 로 만들면 Confluence API가 거부합니다.

팀 운영 시에는 개인 계정 대신 **공용 봇 계정**으로 토큰을 발급하고, Figma는 대상 파일 View 권한, Confluence는 대상 페이지 편집 권한을 부여하세요.

## Figma 기획서 규칙

번역 대상 프레임 이름 앞에 `Trans` 를 붙입니다 (prefix 고정, 파일 내 모든 페이지 탐색).

```
Page 1
├─ Trans홈화면          ← 이 안의 모든 텍스트 수집
├─ Trans상품상세        ← 이것도 수집
├─ #comments_...        ← prefix 없으므로 무시
└─ 디자인메모           ← 무시
```

- 대상 노드 타입: FRAME, GROUP, COMPONENT, COMPONENT_SET, INSTANCE, SECTION
- **페이지 구분 라벨**: 프레임 이름에서 `Trans`를 뗀 나머지 (`Trans홈화면` → "홈화면")
- 같은 텍스트가 여러 프레임에 있으면 1개만 수집 (첫 발견 프레임의 라벨 기록)

## Confluence 표 규칙

- 페이지에서 **"한국어" 컬럼이 있는 첫 번째 표**를 대상으로 삼습니다. 표가 없으면 에러 (자동 생성하지 않음).
- 컬럼 순서·구성은 자유 — 헤더를 동적으로 읽습니다. `페이지 구분` 컬럼이 있으면 프레임 라벨이 채워집니다.
- 지원 언어(5개 고정): **한국어(필수)** + 영어 · 일본어 · 중국어(간체) · 중국어(번체)
- 표에 없는 지원 언어를 체크하면 컬럼이 자동 추가됩니다 (헤더 + 전 행 빈 셀).
- 지원 언어가 아닌 컬럼(비고, 담당자 등)은 **절대 건드리지 않습니다**.

### 헤더 표기 별칭

헤더가 다음 표기여도 같은 언어로 인식합니다 (공백·대소문자 무시):

| 언어 | 인식되는 표기 |
|---|---|
| 한국어 | 한국어, Korean, ko, 국문, kr |
| 영어 | 영어, English, en, 영문 |
| 일본어 | 일본어, Japanese, ja, jp, 일어 |
| 중국어(간체) | 중국어(간체), 간체, 중문(간체), zh-CN, zh-Hans |
| 중국어(번체) | 중국어(번체), 번체, 중문(번체), zh-TW, zh-Hant |

---

## 중복 판정 기준

- **전체 언어 컬럼**(한국어 + 영어·일본어·중국어) 기준 비교 — 영어 원문은 영어 셀과 매칭됨
- 대소문자 무시, 앞뒤 공백 무시, **줄바꿈·연속 공백은 단일 스페이스로 취급**
  (여러 줄 문단도 재실행 시 중복으로 인식됨)
- 이미 표에 있는 용어는 다시 추가하지 않음 — 같은 링크로 몇 번을 실행해도 안전

## 디폴트 번역 정책 (Claude가 채울 때)

| 분류 | 처리 |
|---|---|
| 메뉴/카테고리/일반 용어 (예: 회사소개, 채용) | 일반적 직역 |
| 마케팅 카피/슬로건 (긴 문장·단락) | 자연스러운 **의역** |
| 회사명/브랜드명 (예: 이롬넷, 페이버스) | 영어는 공식 표기 / 일·중은 음역 → **정확성 검수 필요** |
| 업계 약어 (예: PG = Payment Gateway) | **원문 보존** |
| 주소·전화·이메일·사업자번호 | **번역 스킵** (빈 칸 유지) |
| 더미/placeholder 텍스트, 색상 코드 | **번역 스킵** (Figma 정리 권장) |
| 이미 영어인 항목 (예: `Contact Us`) | 영어 컬럼은 그대로 / 일·중만 번역 |

- 정책은 `translator.py`의 시스템 프롬프트에 내장 — 누가 실행해도 동일하게 적용됩니다.
- 기존 표의 번역을 few-shot 재료로 참조하므로, 표를 검수할수록 새 번역 품질이 좋아집니다.
- **내용이 이미 있는 셀은 절대 덮어쓰지 않습니다.** 미체크 언어 컬럼도 건드리지 않습니다.

## 에러 대응

| 증상 | 원인 / 조치 |
|---|---|
| "'한국어' 컬럼이 있는 표를 … 찾지 못했습니다" | 페이지에 한국어 헤더 표가 없음 → 표 생성 후 재시도 (위 별칭 표 참고) |
| "Confluence URL에서 페이지를 찾지 못했습니다" | URL 형식 오류 또는 봇 계정에 페이지 접근 권한 없음. 단축 URL(`/wiki/x/..`)은 접근 권한이 있어야 해석 가능 |
| API 호출 실패 401/403 | 토큰 만료·권한 부족. Scoped 토큰을 쓴 경우도 이 에러 — Classic 토큰으로 재발급 |
| API 호출 실패 409 (동시 편집) | 자동으로 2회 재시도함(500ms 간격). 그래도 실패하면 편집 중인 사람이 끝난 뒤 재실행 |
| 번역이 빈 칸으로 남음 | ① 정책상 스킵 대상 (주소·더미 등 — 정상) ② Claude 응답 파싱 실패 (재실행하면 빈 셀만 다시 시도) |
| 긴 문단 번역이 중간에 끊김 | 드물게 발생 — 검수 시 직접 수정 |

## 주의사항

- `.env` 는 git에 올리지 않습니다 (`.gitignore` 처리됨). 배포 환경은 Streamlit Cloud Secrets만 사용.
- Confluence **Cloud** 전용 (`*.atlassian.net`). Server/Data Center는 엔드포인트가 다릅니다.
- 표를 통째로 지우거나 한국어 컬럼 헤더를 바꾸면 표 인식이 실패합니다.
